streamsx.topology

Write millisecond latency IBM Streams applications entirely in Java, Scala or Python(beta).

Java Application API

1.3.0 release - 2016/05/09

The Java Application API enables a developer to create streaming applications entirely in Java and Scala for IBM Streams. The API employs a functional style of programming -- a developer may define a graph's flow and data manipulation simultaneously. 


    /*
     * Declare a source stream (hw) with String as tuples that sends two
     * tuples "Hello" and "World!", and prints them to output.
     */
    Topology topology = new Topology("HelloWorld");
    TStream<String> hw = topology.strings("Hello", "World!");
    hw.print();
    StreamsContextFactory.getEmbedded().submit(topology).get();

Beta Python Support

1.4.1_beta release - 2016/06/16

This is a beta release of the com.ibm.streamsx.topology SPL toolkit that adds the ability to implement IBM Streams applications in Python.

  • Python Application API - allows an application to be built using Python in a similar approach to the Java Application API, but with Python so that streams are created, processed and sinked using Python functions and tuples are Python objects.
    • Getting Started Guide
  • Python functions as SPL operators - the ability to easily convert Python functions to SPL operators for IBM Streams.
For details see the SPLDOC for com.ibm.streamsx.topology.python namesapce.
